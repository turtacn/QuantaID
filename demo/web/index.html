使命与核心定位

QuantaID 不仅是一个身份认证工具，更是企业数字化转型中身份管理的"基础设施"，通过标准化与可扩展性平衡安全与效率

核心使命

革新企业身份管理模式，提供轻量级、插件化、符合标准的统一认证平台，解决多样化企业环境下身份系统碎片化、定制成本高、集成复杂的核心痛点。

核心差异

  • 摒弃"一刀切"的重型方案，以"配置驱动"替代"定制开发"，降低企业使用门槛
  • 原生支持混合云部署，无缝衔接企业现有IT架构（本地数据中心+公有云+私有云）
  • 以"身份为中心"构建安全边界，而非依赖网络隔离，适配云原生时代的动态环境

  • 支持"集团-子公司"多级权限隔离，基于RBAC实现精细化权限管控

业务价值

用户登录体验提升90%，IT维护成本降低60%，跨系统数据安全合规性显著增强。

金融行业自适应MFA

基于风险动态调整认证强度，平衡安全与体验

核心挑战

金融机构需满足严格的安全合规要求（如等保2.0），但传统"一刀切"的MFA模式（如每次登录都需短信验证）导致高净值客户体验差，同时无法有效识别异常登录风险。

QuantaID 解决方案

  • 基于"设备指纹+地理位置+行为模式"构建风险评分模型
  • 低风险场景（常用设备+常用地点）免MFA，高风险场景强制WebAuthn硬令牌
  • 与交易系统联动，大额转账等敏感操作自动触发强化认证

业务价值

客户投诉率降低40%，异常登录拦截率提升85%，完全满足金融行业合规要求。

云原生环境集成

为K8s、微服务提供统一身份认证与授权

核心挑战

企业云原生转型中，K8s集群、微服务、API网关等组件身份认证方式各异，缺乏统一管控，开发者需维护多套凭证，同时存在"权限过宽"的安全风险。

QuantaID 解决方案

  • 通过OIDC为K8s提供RBAC身份源，实现"人-服务账号"统一管理
  • 为微服务提供JWT令牌认证，集成API网关实现细粒度接口授权
  • 基于Helm Chart快速部署，与云原生监控体系（Prometheus/Grafana）联动

业务价值

开发者效率提升50%，微服务安全漏洞减少70%，云资源权限管控粒度细化至"接口级"。

跨国企业合规管理

适配全球不同地区的数据隐私法规

核心挑战

跨国企业需同时满足欧盟GDPR、美国CCPA、中国个人信息保护法等不同地区法规，身份数据存储、传输、审计要求差异大，传统方案难以适配。

QuantaID 解决方案

  • 支持"区域化数据存储"，身份数据本地化部署，避免跨境传输风险
  • 内置GDPR"被遗忘权"功能，支持用户身份数据一键删除与导出
  • 多语言审计报告，自动适配不同地区合规格式（如ISO27001报告模板）

业务价值

全球合规成本降低55%，避免因法规不合规导致的百万级罚款，提升用户数据信任度。

快速上手部署

提供三种主流部署方式，满足不同场景需求，最快5分钟完成初始化

二进制部署

零依赖，适合快速测试与边缘场景

前提条件

Go 1.21+（可选，仅编译需要）

部署步骤

# 安装QuantaID CLI
go install github.com/turtacn/QuantaID/cmd/qid@latest

# 初始化部署配置
qid init --config-dir ./qid-config

# 启动QuantaID服务
qid server start --config ./qid-config/server.yaml

优势

部署简单、资源占用小、启动速度快

Docker部署

环境一致，适合标准化生产环境

前提条件

Docker 20.10+、Docker Compose 2.0+

部署步骤

# 拉取最新镜像
docker pull quantaid/quantaid:latest

# 下载Docker Compose配置
curl -O https://raw.githubusercontent.com/turtacn/QuantaID/master/deployments/docker-compose.yml

# 启动服务
docker-compose up -d

优势

环境隔离、版本管理方便、易于迁移

Kubernetes部署

云原生，适合大规模高可用场景

前提条件

Kubernetes 1.24+、Helm 3.8+

部署步骤

# 添加QuantaID Helm仓库
helm repo add quantaid https://helm.quantaid.dev
helm repo update

# 安装QuantaID（内置PostgreSQL/Redis）
helm install quantaid quantaid/quantaid \ --set postgresql.enabled=true \ --set redis.enabled=true

优势

自动扩缩容、高可用、运维自动化

基础使用示例（Go SDK）

package main import ( "context" "log" "github.com/turtacn/QuantaID/pkg/client" "github.com/turtacn/QuantaID/pkg/types" ) func main() { // 初始化QuantaID客户端 qid, err := client.New(client.Config{ Endpoint: "https://your-quantaid-instance.com", APIKey: "your-api-key", }) if err != nil { log.Fatal(err) } // 配置OIDC身份提供商 provider := &types.IdentityProvider{ Name: "corporate-sso", Type: "oidc", Enabled: true, Config: map[string]interface{}{ "issuer_url": "https://your-corp-sso.com", "client_id": "quantaid-client", "client_secret": "your-secret", "scopes": []string{"openid", "profile", "email"}, }, } ctx := context.Background() if err := qid.IdentityProviders.Create(ctx, provider); err != nil { log.Fatal(err) } // 生成认证URL，引导用户跳转 authURL, err := qid.Auth.GetAuthorizationURL(ctx, &types.AuthRequest{ Provider: "corporate-sso", RedirectURI: "https://your-app.com/callback", State: "random-state-string", }) if err != nil { log.Fatal(err) } log.Printf("Redirect user to: %s", authURL) }
